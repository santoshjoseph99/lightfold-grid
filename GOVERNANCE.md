# Governance

Lightfold Grid is currently maintained under a lightweight maintainer-led model suited
to an experimental developer alpha.

## Roles

**Contributors** propose changes, report issues, improve documentation, and participate
in technical discussion.

**Maintainers** review and merge changes, triage reports, make releases, manage project
infrastructure, and enforce community and security policies.

**Release maintainers** are maintainers explicitly trusted to create signed releases,
manage release credentials, and coordinate vulnerability fixes. This role may be empty
until public release infrastructure exists.

## Decisions

- Routine changes are accepted through reviewed pull requests.
- Maintainers seek rough consensus for architectural, protocol, security, governance,
  and roadmap changes.
- When consensus is not practical, the active maintainers make the decision and record
  the reasoning in an issue, pull request, or governance update.
- Security embargo decisions may be made privately until coordinated disclosure.

At least one maintainer approval and passing required checks are needed to merge.
Changes to the wire protocol, trust boundaries, release process, or governance require
explicit maintainer review.

## Maintainer Changes

Contributors may be invited to become maintainers after sustained, trustworthy
contributions and sound technical judgment. Existing maintainers decide by consensus.
Maintainers who are inactive for six months may be moved to emeritus status after an
attempt to contact them.

## Releases

Release maintainers decide when a version satisfies its documented release criteria.
Releases must identify known limitations, pass required checks, and follow the security
and release procedures documented in the repository.

## Amendments

Governance changes follow the same public pull-request process as other project
changes. Material changes should remain open for community comment before merging.
